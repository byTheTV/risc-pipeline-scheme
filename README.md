\### risc pipeline scheme with processing of all hazards





\- \*\*Data Hazard (RAW) — Решается через FORWARDING (Проброс):\*\*

&#x20;   - \*\*Как:\*\* Есть блок `Forward Unit` внизу. Он смотрит на регистры. Есть линии обратной связи от `EX/MEM` и `MEM/WB`, которые идут назад к MUX-ам перед ALU.

&#x20;   - \*\*Результат:\*\* Если инструкция хочет данные, которые еще не записаны в регистры, они берутся напрямую из конвейера. Пузырей нет.

\- \*\*Load-Use Hazard (Чтение из памяти) — Решается через STALL (Задержка):\*\*

&#x20;   - \*\*Как:\*\* Блок `Hazard Unit` (сверху) отслеживает ситуацию, когда `lw` (загрузка) стоит перед инструкцией, использующей эти данные. Он подает сигнал `STALL` на регистры `IF/ID` и `ID/EX`.

&#x20;   - \*\*Результат:\*\* Конвейер замирает на 1 такт, давая данным время дойти из памяти.

\- \*\*Control Hazard (Ветвление) — Решается через FLUSH (Очистка):\*\*

&#x20;   - \*\*Как:\*\* Есть `Comparator` (сравнение регистров для ветки) и сигнал `Branch taken`. Если ветка берется, сигнал `FLUSH` очищает регистр `IF/ID`.

&#x20;   - \*\*Результат:\*\* Ненужные инструкции, которые успели залететь в конвейер, превращаются в `NOP` (пузыри) и не выполняются.
